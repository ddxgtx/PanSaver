# PanSave 项目开发规则与指南 (agent.md)

本文件定义了 PanSave 项目的设计原则、编码标准、安全约束和技术细节，开发本项目的 Agent 必须严格遵守以下规则。

---

## 1. 架构与设计原则

1. **统一的网盘适配层**：
   - 所有与百度网盘的交互（登录验证、容量获取、分享解析、文件转存、重命名等）必须封装在 `app/providers/baidu/` 中。
   - 业务逻辑服务（如 `transfer_service.py`）不得直接依赖 `baidupcs-py` 的底层实现，必须通过统一的 `BaiduPCSClient` 适配层进行操作。
2. **轻量级依赖**：
   - 数据库交互采用 Python 标准库 `sqlite3`，以保持轻量和易于 PyInstaller 单文件打包。
   - 避免引入繁重的 ORM 框架，确保 SQL 语句清晰易懂并防止依赖冲突。

---

## 2. 安全与隐私规则

1. **严禁保存明文敏感信息**：
   - 用户的百度网盘 Cookie（BDUSS、STOKEN）和分享链接的提取码必须加密存储。
   - 加密密钥通过用户首次运行系统时创建的管理员密码（通过 PBKDF2）动态派生，或在本地生成随机密钥文件，绝不能将硬编码密钥写入代码。
2. **敏感信息日志脱敏**：
   - 在所有的日志输出中，严禁打印完整的 Cookie、提取码、管理员密码或加密密钥。
   - 任何敏感字段的打印应进行掩码处理（例如：`BDUSS: BDUS***...***`）。
3. **访问控制**：
   - 桌面模式下，Flask 接口默认只能绑定 `127.0.0.1`。
   - 仅在用户主动开启“允许远程访问”设置后，方可绑定 `0.0.0.0`。
4. **数据导出安全**：
   - 当用户请求备份或导出任务配置时，必须提供“排除敏感字段”选项，默认勾选不导出加密凭证。

---

## 3. 数据库与存储规范

1. **数据库连接管理**：
   - 使用 SQLite 存储，文件路径配置为可选项（桌面版使用系统应用数据目录，如 `%APPDATA%/PanSave/`；Docker/Linux 版使用 `./data/` 目录）。
   - 编写 `db.py` 统一管理数据库连接与游标。应在多线程环境下使用合适的连接锁定机制，防止 SQLite 的 `database is locked` 错误。
2. **迁移机制**：
   - 提供简易的数据库初始化与结构版本校验脚本，当发现数据库结构版本低于当前代码要求时自动执行轻量级更新（如 `ALTER TABLE`）。

---

## 4. 异常处理与指数退避重试

对于转存或订阅解析过程中遇到的异常，必须按照以下分类进行处理：

| 异常类型 | 处理策略 |
| :--- | :--- |
| **可恢复异常**（如网络超时、HTTP 502/503、网盘接口频控） | 采用指数退避机制重试（第 1 次 5s，第 2 次 15s，第 3 次 60s），最终失败后记录任务状态并发送通知。 |
| **不可恢复异常**（如分享链接失效、提取码错误、Cookie 过期） | 立即终止任务，标记状态为失败，暂停相关调度，并发送紧急警报通知，不进行重复重试。 |
| **网盘容量不足** | 立即终止转存，发送容量预警通知。 |

---

## 5. 安全覆盖与事务一致性

在执行“安全覆盖 (safe_overwrite)”策略时，必须严格执行事务性多步操作：

1. **临时转存**：首先将分享资源完整转存到隐藏临时目录 `/自动转存/.pansave/staging/task_<id>/run_<run_id>`。
2. **完整性校验**：检查临时目录中的文件数量和总大小是否与源快照完全一致。
3. **备份原目录**：将目标正式目录移动到备份文件夹 `/自动转存/.pansave/backup/<task_name>_<timestamp>`。
4. **移入正式目录**：将临时目录重命名/移动到目标正式路径。
5. **回滚机制**：如果步骤 1-4 中有任何一步失败，必须立即终止，并尝试将备份目录恢复回正式目录，向用户发送回滚报告。
6. **历史清理**：操作成功后，仅保留配置的最近 N 个历史备份，其余自动清理。

---

## 6. 定时调度规则

1. **并发控制**：
   - APScheduler 定时任务必须配置 `max_instances=1`，确保同一任务同一时间禁止并发运行。
2. **运行上下文记录**：
   - 每次任务运行前，必须分配唯一的 `run_id`，所有转存操作、差异计算、通知结果以及每一条 Loguru 写入都必须关联此 `run_id`，以支持日志筛选与错误排查。

---

## 7. 前端与界面规范

1. **现代而动感的 UI 设计**：
   - 前端采用 Vue 3 + TypeScript + Element Plus。
   - 统一采用现代深色模式或高阶渐变主题，减少刺眼的白底。
   - 所有按钮、卡片需添加平滑的过渡效果（`transition`）。
2. **无占位符**：
   - 严禁使用静态占位图片，如有必要必须使用 `generate_image` 工具生成真实 UI 配图或切图。
3. **接口健壮性**：
   - 前端所有 API 请求应配备全局异常捕获，对 401（未登录）、500（系统错误）做出友好提示，防止页面假死。
